# What Replaces vs What Preserves

## Replaces
- 11 dead tracking databases (zero data, pure schema weight)
- `habitat-bootstrap` L3 layer (SQLite learned_patterns query)
- Manual session note reading for trajectory
- Per-service probing for trap state
- `memory-federate` proposed S101 script (STDB is the unified query surface)

## Preserves
- **Auto-Memory** (MEMORY.md + *.md) — human-curated, always loaded
- **Obsidian vault** (215 notes) — human-authored, canonical
- **POVM Engine** (3554 pathways) — dual-write, never deleted
- **RM Engine** (TSV heartbeat) — continues as-is
- **CLAUDE.md / CLAUDE.local.md** — project instructions, session state
- **`habitat-bootstrap`** — preserved as fallback

## Evolves
- `habitat-bootstrap` gains L7-L10 sourced from STDB
- ORAC SessionStart hook gains STDB-powered injection
- Pattern reinforcement becomes live, queryable, decaying (not write-once-read-never)
